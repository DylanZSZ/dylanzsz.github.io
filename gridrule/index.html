Interactive ARC-AGI-3-style environment · label-free RL · ≤1.5B scale

Can a tiny model learn to decompose problems
by proposing its own subgoals?

We built GridRule — a cheap, procedural gridworld that captures the two hard demands of ARC-AGI-3: latent-rule induction by trial and error, and compose-back subgoal chaining. We then asked whether a 0.8B model trained with label-free self-distillation, using nothing but environment feedback, can learn a transferable decomposition skill by proposing the subgoals it wants to reach.

A note before you read

This is a pilot endeavor — a small-scale, exploratory study conducted by a single researcher on consumer hardware. The findings are encouraging but preliminary: sample sizes are small (n=2–3 seeds for most claims), the environment is synthetic, and we make no claims about scaling to real ARC-AGI-3. We share this honestly, in the spirit of open, incremental science. Treat it as an early conversation starter, not a finished result.

TL;DR — pilot findings
  • Self-proposed subgoals help. A 0.8B model trained with nothing but environment reward and label-free self-distillation, using coordinate subgoals it proposes itself, reaches ≈1.8× baseline pass@1 on a held-out test set — reproduced on two independent seeds.
  • Decomposition transfers to unseen, harder stages. Trained only on primitive single-gate rooms, the subgoal policy solves unseen two-gate compositions at 0.94 pass@1 (vs baseline 0.58) — and the quality is so high it exceeds policies trained directly on compositions.
  • The bottleneck is execution, not decomposition. When the environment auto-navigates to each proposed subgoal (oracle execution), scores jump dramatically: unseen 3-gate tasks go from 0.26 → 0.80+. The model's decomposition strategy transfers further than its navigation skill.
  • Three cheap tricks compound the gains further: LoRA-souping (average weights across seeds), diversity-aware filtering of distillation data, and a tuned promotion bar — each independently validated.
  • Harder mechanics reveal distinct failure modes. Irreversibility breaks decomposition (the model proposes a trap); hazards break execution; crafting breaks sequencing. Oracle-execution probes cleanly separate them.
The environment

GridRule: a knobbed proxy for ARC-AGI-3

ARC-AGI-3 asks solvers to induce hidden rules by trial and error and chain sub-solutions into multi-step plans. The real benchmark is visual, API-bound, and expensive to iterate on. GridRule is a cheap, deterministic, text-renderable proxy that captures both demands in a fully procedural generator — every episode is fresh, every control and object role is latently permuted, and a BFS oracle is always available for reward.

-- Episode render (coords view) ---------------------------- Room 0 (start): key K₁ at (2,3) Agent at (1,1) ← controls are hidden; must be induced Room 1 (locked): door needs K₁ ← binding hidden; learned by bumping switch S₂ at (3,4) Room 2 (gated): gate needs S₂ toggled EXIT -- Subgoal proposal (model output) ------------------------- SUBGOAL:(2,3) ← pick up key first env verifies; hindsight distills if reached even on failure

The key design choices: controls are latently permuted per episode (which action goes which direction), object roles are latently permuted (which glyph is a key vs a decoy), and key-door bindings are per-episode random. The solver cannot memorize its way through — it must induce the rules from experience on every episode.

The curriculum

Stages 0–2 introduce each mechanic as a single gate (s0: door, s1: gate, s2: push). Stages 3–4 compose two mechanics. Stages 5–6 add room size and chain length. A promotion bar (we use 0.65 pass@1) gates advancement — the key hyperparameter, as we'll see.

The method

CTTD + self-proposed subgoals

The training loop is CTTD (Continual Test-Time self-Distillation): a four-phase cycle that lets the model teach itself from its own successful episodes, with no human labels anywhere.

  • W
    WAKE

    Roll out K episodes per level. The model proposes a subgoal SUBGOAL:(r,c), then acts. The environment verifies whether the proposed cell was reached.

  • R
    REWARD

    Score each episode with environment success and action efficiency (RHAE = min(1, optimal/used)²). No oracle labels — the only signal is whether the model won and how efficiently.

  • F
    FILTER

    Keep only efficient successes (≤2.5× optimal). Optionally cap duplicate behavioral signatures (--diverse-filter) so one lucky solution mode can't flood the distillation set.

  • D
    DREAM

    LoRA-SFT on kept episodes + replayed buffer. EMA weight averaging (α=0.7) smooths the policy. Advance to next curriculum stage when the promotion bar is met on a held-out eval set.

Self-proposed subgoals with hindsight

The key addition over vanilla CTTD: at each step, the model emits SUBGOAL:(r,c) — a coordinate it commits to reaching next. The environment only verifies the proposal (was that cell reached?); it never tells the model what a good subgoal would be. An anti-hack distance guard prevents trivially proposing the current cell.

Hindsight distillation: if an episode fails overall but the model did reach its proposed subgoal, we distill the prefix up to the subgoal-achievement step. This recycles "partial credit" episodes that would otherwise be discarded — the model learns this decomposition step was correct even if the rest went wrong.

Why coordinate subgoals? An early experiment tried typed/abstract labels (ITEM / GATE / EXIT). At 0.8B, abstract naming gave ~0 bootstrapping wins — an abstract name provides no navigation anchor. Emitting a coordinate forces the model to localize a target, doing double duty as decomposition and perception. Abstraction likely needs a warm-start from a competent coordinate policy first.

The strong result is not a fluke: seed-1 (0.699) and seed-2 (0.679) both reach ≈1.8× baseline pass@1, independently. The tuned promotion bar (0.65) guarantees ≥ baseline as a floor — no bar-0.65 run is below baseline — while seed variance still governs whether the strong ceiling is hit.

The promotion bar is the key hyperparameter

Our fair evaluation revealed that curriculum racing — advancing stages quickly — anti-correlates with policy quality. The cause: the promotion bar controls how thoroughly the model masters each primitive before advancing.

The sweet spot ≈ 0.65: achievable enough to advance, strict enough to force per-stage mastery. Too low races to weak policies; too high stalls entirely.

Transfer & compositional generalization

Master primitives → compose

The most exciting finding: a policy trained only on single-gate primitive rooms solves unseen two-gate compositions better than a policy directly trained on those compositions.

Transfer to unseen harder stages (≈1.7×, replicated)

We capped all arms at stage 0 (single-door primitive), then evaluated on unseen stages 1–4. Self-proposed subgoals transfer robustly:

Arm (all trained s0 only)Unseen s1–4 pass@1vs baseline
Plain baseline0.264—
Subgoal + hindsight (2 seeds, mean)0.465+1.76×
Subgoal − hindsight (3 seeds, mean)0.440+1.67×

All 5 subgoal arms (across 5 independent seeds) land above baseline. The transfer is real, consistent, and present regardless of whether hindsight is used. This is the most robustly replicated finding in this pilot.

A side note on hindsight: a single-seed ablation suggested hindsight hurts transfer. The second seed flipped the sign. At n=2, hindsight's effect on transfer is within seed variance — we can't make a confident claim either way. The consistent effect is that −hindsight achieves slightly better trained-stage mastery, while ±hindsight are roughly tied on transfer.

The "master parts" lesson

The subgoal policy, having only seen each mechanic as a single gate, scores 0.944 on unseen two-door and 0.725 on unseen two-mix. These numbers are higher than the full-curriculum subgoal policy that actually trained on s3/s4 (0.744/0.550). The right curriculum is "master the parts," not "grind the whole."

Mechanism

Decomposition transfers — execution is the bottleneck

To separate planning from navigation, we built an oracle-execution probe: the environment auto-navigates (via BFS) to whatever subgoal the model proposes, then the model proposes the next one. The model only decomposes; execution is perfect. What happens?

StageNormal pass@1Oracle-exec pass@1Lift
s0 1-door0.7381.000+0.26
s1 1-gate0.6940.950+0.26
s2 1-mix0.6690.750+0.08
s3 2-door0.7440.950+0.21
s6 3-gate (unseen)0.2630.80+0.54
Replicated on two seeds

On s6 (3-gate, never trained on): seed-2 normal→oracle is 0.26→0.80; seed-3 is 0.38→0.95. The model's decomposition for unseen 3-gate tasks is near-perfect. What collapses performance is the inability to reliably navigate to the proposed waypoints over long chains — an execution failure, not a planning failure.

Transfers well ✓

Decomposition — which subgoal to pursue next. Proposal sensibility is 2.5–5× above chance on every unseen stage, and does not degrade on 3-gate tasks.

Execution bottleneck ✗

Multi-step navigation to the proposed waypoint. More cells, more turns, more chances to take a wrong step. Bigger models help here (1.7B vs 0.8B lifts the hardest stage by +0.11).

Probe sensibility at depth

We measure whether proposed subgoals land on BFS-optimal paths. On unseen compositions, proposal sensibility is as high or higher than on trained stages (s3 0.83, s4 0.65 vs chance ≈0.24 — vs trained s1/s2 at 0.52/0.59). The model generalizes its planning to new compositions without degradation. The bottleneck is purely navigational.

Three cheap compound gains

LoRA-souping, diverse filtering, capacity

LoRA-souping: remove seed-selection for free

The strong outcome (0.68–0.70 pass@1) is seed-dependent. Rather than picking the best seed post-hoc, we average the LoRA weights of 3 trained adapters (model souping) and evaluate the averaged model — no new training.

The soup scores 0.506 on unseen s1–s4, above the best individual seed (0.491) and the 3-seed mean (0.440). It eliminates the seed-selection problem and lifts transfer — for free.

Diversity-aware filtering: better distillation data

When many winning episodes share the same behavioral trajectory (same event sequence), the distillation set collapses onto one solution mode. Capping how many duplicates we keep per signature (--diverse-filter) forces the SFT data to be behaviorally diverse.

Both metrics improve on both seeds — not a trade-off between pass@1 and pass@8. The filter is cheap and drop-in.

Cross-domain

Few-shot acceleration on a novel environment

We built a second environment — SeqGrid — with the same coordinate/action/subgoal interface as GridRule but a completely different rule: the exit opens only after stepping on pads in a hidden order (vs lock-and-key). A GridRule-trained subgoal adapter had never seen this mechanic. What happens when it is fine-tuned on SeqGrid with the same self-distillation loop?

Base (cold start)GridRule-warm start
Primitive pad task (q0): pass@1 at round 00.21–0.360.86 (immediate)
Novel 2-pad rule (q1): final pass@10.2420.386 (+1.59×)
Novel 2-pad rule (q1): pass@80.710.76 (smaller gap)

The warm-started model learns the transferable primitive 3–4× faster (immediate competence vs 6–8 rounds) and reaches 1.59× higher final pass@1 on the novel rule. The pass@8 gap is smaller — with enough retries, cold start catches up. The warm-start advantage is concentrated in first-try reliability.

Replicated on 2 seeds. Both warm seeds outperform both cold seeds on q1 pass@1, with clean separation. The cross-domain story is: learning GridRule decomposition accelerates learning a new environment's rule — not that cold start can't eventually learn it.

Three hard mechanics, three distinct failure modes

Oracle-execution probes cleanly diagnose what binds in each case:

Irreversibility flips the bottleneck (replicated, n=2)

Under reversible mechanics, oracle-execution rescues nearly everything (s3 0.744→0.950 with oracle). Under irreversibility on the 1-door stage, oracle-exec provides zero lift — the model is proposing the decoy cell (16–22% of episodes), and perfect execution walks it straight into the trap. Execution isn't the problem when the proposed plan is already wrong. This is the most interesting frontier finding: a mechanic that specifically targets the decomposition faculty.

Honest caveats

What this pilot can and cannot claim

We want to be straightforward about the limitations of this work. This is a pilot, and we think that framing matters.

  • !
    Small n.

    Most claims are based on n=2–3 seeds. The strong result (0.68–0.70 pass@1) is reproduced on two seeds, which is meaningful, but far from a rigorous statistical claim. The ceiling is still seed-variable; bar-0.65 reliably floors at baseline but doesn't guarantee the strong outcome.

  • !
    Synthetic environment.

    GridRule is a proxy, not the real ARC-AGI-3. It captures the demands we care about (latent rule induction, compose-back decomposition) in a convenient package, but we cannot claim the findings transfer to visual, high-dimensional, or open-ended settings.

  • !
    Frontier mechanics at n=1.

    Most results in the "harder mechanics" section are single-seed. The three-failure-mode taxonomy is promising but should be treated as a hypothesis to replicate, not an established result.

  • !
    Seed variance remains.

    We observed seed-2 on bar-0.65 reach 0.679 while seed-0 on the same bar reached only 0.378. We do not yet have a reliable predictor of which seed will produce the strong outcome. LoRA-souping partially addresses this but doesn't eliminate it.

None of these caveats, we think, undermine the core finding: self-proposed coordinate subgoals, in this setting, reliably improve transfer and compositional generalization, replicated at n=2. The pilot is a starting point. We share it to invite scrutiny and collaboration, not to claim a solved problem.

Takeaways

What this pilot suggests

Key finding #1

Self-proposed coordinate subgoals let a 0.8B model learn a transferable decomposition strategy without labels — replicated at n=2.

Key finding #2

The bottleneck is execution, not planning. The model's decomposition transfers to depth-3 unseen tasks; navigation is what fails under longer chains.

Key finding #3

Master parts, then compose. Training on primitives and evaluating on compositions outperforms training directly on compositions — at this scale.

The lesson we take from the irreversibility finding is the one we find most broadly interesting: environment mechanics can selectively target either the decomposition or execution faculty. This suggests a path toward diagnostic benchmarks that measure not just whether a model solves a task, but which cognitive component is the limiting factor. We hope GridRule's knob-based design makes it a useful tool for that kind of probing, even beyond the specific method we tested here.

We're happy to share the environment and training code. If you find GridRule useful as a testbed — or if you see something wrong with our analysis — please reach out.
